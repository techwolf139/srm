# 合同审计（中文版）

## 概述

**专业合同审查需要对风险、清晰度和可执行性进行系统性分析。**

本skill提供结构化的合同审计方法，基于《中华人民共和国民法典》合同编，结合中国法律实践，精确定位高风险内容。

**法律依据：**
- 《中华人民共和国民法典》合同编
- 《最高人民法院关于适用〈民法典〉合同编通则若干问题的解释》

## 适用场景

**适用于以下合同类型：**
- 买卖合同、租赁合同、借款合同
- 承揽合同、技术合同、建设工程合同
- 服务协议（SaaS、外包、咨询）
- 采购合同、保密协议（NDA）
- 劳动合同、承包商协议
- 合伙合同、保证合同
- 许可协议、任何需要风险评估的法律文件

## 审计框架

### 第一阶段：合同类型识别

```
识别合同类型 → 确定适用法律条款 → 选择审查重点
```

### 第二阶段：主体资格审查

```
1. 合同主体是否具备民事行为能力
2. 代理人是否有有效授权
3. 法人分支机构签约是否有效
4. 多个主体权利义务是否明确
```

### 第三阶段：形式要件审查

```
1. 合同是否采用书面形式
2. 签字盖章是否完整
3. 日期是否明确
4. 附件引用是否完整
```

### 第四阶段：必备条款检查

**《民法典》第469条要求的书面合同要素：**
```
当事人、标的、数量、质量、价款、履行期限、地点方式、违约责任、争议解决
```

**逐项核查是否存在、是否清晰。**

### 第五阶段：风险分级评估

**按风险等级对每一条款分类：**

| 等级 | 分类 | 行动 |
|------|------|------|
| 🔴 严重 | 违反法律强制性规定、显失公平 | 需立即重新谈判或拒绝 |
| 🟠 高风险 | 权利不对等、模糊条款、潜在纠纷 | 建议协商修改 |
| 🟡 中等 | 缺失保护、定义不清 | 需要澄清 |
| 🟢 低风险 | 行业标准条款、权利义务平衡 | 可接受 |

### 第六阶段：高风险内容检测

#### 🔴 严重红旗（违反《民法典》强制性规定）

```
- 违反法律强制性规定的条款（民法典第153条）
- 损害社会公共利益的条款
- 格式条款免除己方责任、加重对方责任（民法典第497条）
- 造成对方财产损失的欺诈、胁迫条款
- 显失公平的条款（民法典第151条）
- 重大误解未及时行使撤销权
```

#### 🟠 高风险模式

```
【权利义务不对等】
- 单方终止权、无理由解除权
- 单方面修改合同条款权
- 无限责任或责任无上限
- 未定义的"最大努力"/"合理努力"

【履约风险】
- 履行期限约定不明
- 质量标准不具体
- 付款条件不清晰
- 验收标准缺失

【知识产权风险】
- 知识产权归属模糊
- 职务成果归属约定不明
- 保密义务范围过宽

【争议解决风险】
- 管辖条款明显不利
- 仲裁条款无效风险
- 适用法律选择风险

【特殊风险】
- 自动续约无充分通知期
- 最低消费/不可退还条款
- 违约金过高（超过实际损失30%可能被调减）
```

### 第七阶段：合法性审查

**《民法典》重点条款审查：**

| 问题类型 | 法条依据 | 审查要点 |
|----------|----------|----------|
| 合同无效 | 第153条 | 是否违反法律强制性规定 |
| 可撤销 | 第147-151条 | 重大误解、欺诈、胁迫、显失公平 |
| 格式条款 | 第496-498条 | 是否尽到提示说明义务 |
| 合同解除 | 第563条 | 解除条件是否符合法定情形 |
| 违约金 | 第585条 | 是否过高可能被调减 |
| 保证责任 | 第686条 | 保证方式是否明确（一般保证vs连带责任） |

### 第八阶段：报告生成

```markdown
# 合同审查意见书

## 一、合同基本信息
- 合同名称：[合同名称]
- 合同各方：[各方名称]
- 签订日期：[日期]
- 合同类型：[类型识别]
- 审查日期：[日期]

## 二、审查依据
- 《中华人民共和国民法典》合同编
- 相关司法解释

## 三、审查发现

[列出违反法律强制性规定的条款]

### （二）完整性问题
[列出缺失的必备条款]

### （三）风险提示

#### 🔴 严重问题
| # | 位置 | 条款 | 风险 | 法律依据 | 建议 |
|---|------|------|------|----------|------|
| C1 | ... | ... | ... | ... | ... |

#### 🟠 高风险问题
| # | 位置 | 条款 | 风险 | 建议 |
|---|------|------|------|------|
| H1 | ... | ... | ... | ... |

#### 🟡 中等问题
| # | 位置 | 条款 | 风险 | 建议 |
|---|------|------|------|------|

## 四、修改建议

### 条款优化建议示例：

**违约金条款优化：**
原条款："违约金为合同总额的20%"
建议："违约金为实际损失的30%，最高不超过合同总额的15%"

**争议解决条款优化：**
原条款："由甲方所在地法院管辖"

## 五、结论
[综合审查结论：批准/需修改后批准/不建议签署]

## 六、声明
**免责声明：** 本审查意见仅供辅助参考，不构成正式法律意见。对于重大合同，建议咨询专业律师。
```

## 商事合同专项审查要点

### 买卖合同
- 所有权转移时间（民法典第224条）
- 风险承担转移（民法典第606条）
- 质量异议期（民法典第621条）

### 租赁合同
- 租赁期限（民法典第707条，不超过20年）
- 维修责任（民法典第713条）
- 优先购买权（民法典第726条）

### 借款合同
- 利息约定（不超过合同成立时LPR四倍）
- 担保措施有效性
- 还款计划明确性

### 承揽合同
- 工作成果验收标准
- 报酬支付条件
- 定作人解除权（民法典第787条）

### 技术合同
- 知识产权归属（民法典第859条）
- 技术成果分享比例
- 保密义务范围

### 劳动合同
- 试用期合法性
- 工资支付时间
- 社会保险缴纳
- 解除条件合法性

## 行业特定审查要点

### 金融行业
- 利率合规性
- 担保登记要求
- 投资者适当性

### 房地产行业
- 预售许可核查
- 面积差异处理
- 建设工程资质

### 互联网行业
- 个人信息保护（个保法）
- 数据跨境传输合规
- 格式条款显著提示

### 外贸行业
- 贸易术语（Incoterms）
- 报关责任约定
- CISG适用选择

## 位置标识协议

**关键：始终标注具体位置。**

```
格式：[章节编号]，[段落/句子]，[行/页（如适用）]
```

## 常见错误

| 错误 | 问题 | 纠正 |
|------|------|------|
| 只看明显部分 | 风险藏在定义、附件中 | 审查全文档包括标题 |
| 忽略定义条款 | 定义控制解释 | 交叉引用所有定义术语 |
| 遗漏修正案 | 原件+修正案=实际协议 | 作为统一文档阅读 |
| 假设标准语言 | "标准"≠"有利" | 独立评估每一条款 |
| 未标注位置 | 发现无法验证 | 始终标注章节、段落、行号 |
| 忽视民法典新规 | 2021年后保证方式推定变更 | 注意一般保证vs连带责任 |

## 快速检查清单

**基础审查：**
- [ ] 合同主体资格有效
- [ ] 书面形式完整
- [ ] 八大必备条款齐全
- [ ] 签字盖章有效

**风险审查：**
- [ ] 责任条款有上限
- [ ] 终止权对等
- [ ] 违约金合理（不超过损失30%）
- [ ] 争议解决条款公平
- [ ] 不可抗力条款明确
- [ ] 知识产权归属清晰
- [ ] 保密义务范围合理

**合规审查：**
- [ ] 格式条款尽到提示义务
- [ ] 保证方式明确约定
- [ ] 解除条件符合法定情形

## 风险放大因素（需加强审查）

- 合同金额>$100K
- 交易对手为唯一来源
- 多年承诺
- 涉及个人数据
- 跨境元素
- 新型或复杂事项
- 行业监管要求严格

## 合同财务核算分析

### 关键数值提取

**自动识别的数值类型：**

| 类型 | 识别模式 | 示例 |
|------|----------|------|
| 人民币金额 | ¥、RMB、人民币、元 | ¥100,000、50万元 |
| 美元金额 | $、USD | $50,000 |
| 百分比 | %、百分之 | 30%、百分之二十 |
| 违约金比例 | 违约金+百分比 | 违约金为20% |

### 财务风险检测

```python
from docx_helpers import analyze_financial_terms, create_financial_summary

doc = read_docx('合同.docx')
analysis = analyze_financial_terms(doc)

# 分析结果包含:
# - total_mentioned: 所有提及金额的总和
# - currencies: 各币种分布
# - amount_count: 金额提及次数
# - high_amounts: 大额条款(>100万)
# - flagged_unlimited: 是否存在无限责任表述
```

### 财务核算检查清单

**金额条款审查：**
- [ ] 合同总金额是否明确
- [ ] 分期付款是否有明确时间表
- [ ] 预付款比例是否合理
- [ ] 尾款/质保金条款是否清晰
- [ ] 汇率风险是否约定

**费用条款审查：**
- [ ] 违约金计算方式是否明确
- [ ] 违约金是否有上限（建议不超过实际损失30%）
- [ ] 迟延履行利息如何计算
- [ ] 是否有隐藏费用条款
- [ ] 税费是否标明（含税/不含税）

**经济影响评估：**
```
【高风险数值模式】
- 无限责任/无上限赔偿
- 固定金额违约金远高于行业水平
- 百分比违约金超过30%
- 未约定最高赔偿限额
- 自动续约且无价格调整机制

【需核实数值】
- 履约保证金金额是否合理
- 罚款条款是否有明确计算基数
- 提前解除的违约金计算方式
```

## Word文档(.docx)处理

**安装：`pip install python-docx`**

### 快速使用

```python
from docx_helpers import read_docx, detect_risk_keywords, analyze_financial_terms, create_audit_report

doc = read_docx('合同.docx')

# 风险检测
risks = detect_risk_keywords(doc)

# 财务分析
finances = analyze_financial_terms(doc)
print(f"涉及总金额: {finances['total_mentioned']:,.2f} 元")

# 生成报告
create_audit_report('审计报告', findings, '报告.docx')
```

### 辅助脚本

**`docx_helpers.py`** 提供以下功能：

| 函数 | 功能 |
|------|------|
| `read_docx(filepath)` | 读取docx文件 |
| `extract_all_text(doc)` | 提取纯文本 |
| `extract_tables(doc)` | 提取表格数据 |
| `detect_risk_keywords(doc)` | 检测风险关键词 |
| `analyze_financial_terms(doc)` | 提取合同金额、百分比进行财务分析 |
| `create_financial_summary(doc)` | 生成财务分析摘要文本 |
| `create_audit_report(title, findings, output)` | 生成审计报告 |

### 基础操作

```python
from docx import Document

doc = Document('合同.docx')
for para in doc.paragraphs:
    print(para.text)

doc = Document()
doc.add_heading('标题', level=1)
doc.add_paragraph('段落文本')
doc.save('output.docx')
```
